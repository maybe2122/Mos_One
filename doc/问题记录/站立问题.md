# 站立问题:仿真 12 N·m 能站起来,真机力矩不足站不起来

> 状态:**排查中** ｜ 创建:2026-06 ｜ 更新:2026-06-11
> 关联:[`doc/dynamics_gear_ratio_analysis.md`](../dynamics_gear_ratio_analysis.md) ·
> [`todo.md` §力矩/电流不足专项](../../todo.md) · `deploy/mujoco/standup_torque.py` ·
> `motor_control/scripts/{motor_web,robot_web}.py`

---

## 1. 现象

- 仿真(Isaac / MuJoCo)中 `effort_limit_sim = 12 N·m` 即可完成蹲→站并维持站立;
- 真机(GO-M8010-6 @24V,减速比 6.33,整机标称 ≈11.70 kg)撑起吃力、站姿下垂、
  抗扰差,表现为「力矩/电流不足」。

## 2. 已有定量结论(排查前先读,避免重复劳动)

来自 `deploy/common/dynamics.py` + `doc/dynamics_gear_ratio_analysis.md`(2026-06-09):

| 工况 | 理论关节力矩需求 | 对比 |
|---|---|---|
| 静立(4 腿均摊) | τ_knee ≈ **2.2 N·m** | 远小于 12,静立本不该吃力 |
| trot 支撑(2 腿) | ≈ **4.5 N·m** | 仍有余量 |
| 深蹲姿态(h=0.256) | ≈ **14 N·m** | **已超 effort_limit 12!** |
| 动态蹬地 | τ_hip ≈ 12 N·m | 贴上限 |
| 电机峰值(关节侧) | 23.7 N·m | 力矩余量 1.78× |

两个直接推论:

1. **减速比 6.33 已接近最优(N\*=6.16),「换更大减速比」不是解**——选型分析已排除;
2. **起身过程的最低点(深蹲区)力矩需求 ≈14 N·m,本身就高于站定后的 6 倍**。
   "站不起来"未必是电机不行,可能是**起身轨迹经过了力臂最大的深蹲区**——
   仿真里能站起来恰恰说明 12 N·m 是卡着需求线过的,真机任何一点损耗
   (效率、摩擦、电流限幅、电压掉落)都会把它压到线下。

## 3. 假设清单(按「先软后硬、先便宜后贵」排序)

### H1 驱动链路限流/掉压(先查,最可能)

电机本体没问题,但实际到轴的力矩被链路打了折:

- 驱动器**电流上限/力矩上限参数**被设低(出厂保守默认);
- **母线电压掉压**:电池/电源/分电板/串联线路在大负载瞬间掉压限流
  (多电机同时大电流时尤其明显——起身正是 12 个电机同时发力的工况);
- 线阻/接插件压降。

**判定方法**:
1. `motor_web.py` 前馈力矩模式,**单电机吊空**逐档下发 2/4/8/12 N·m(关节侧),
   读 FB 实测力矩——单电机能到额定,说明电机+驱动器参数 OK;
2. 整机做站立动作时,**万用表/示波器盯母线电压**:若发力瞬间从 24V 掉到
   21V 以下,就是供电问题(电池内阻/分电板/线径);
3. 读驱动器电流限制寄存器,与 GO-M8010-6 额定电流对比,被设低则逐步上调。

**判据**:单电机吊空达不到额定 → 驱动器参数问题;单电机 OK 但整机发力时
母线掉压 → 供电问题;两者都 OK → 进 H2。

### H2 控制/换算代码错误

下发的"力矩/位置目标"在换算链上错了一个系数,典型错法:

- **力矩单位侧别搞反**:下发 τ 是转子侧还是关节侧?差减速比 6.33 倍;
- Kt(力矩常数)/减速比标定值不对,下发力矩 → 电流换算错;
- PD 增益单位错(rad vs rot)、方向符号错(个别关节反向发力,互相抵消);
- 站立插值限速过保守,长时间停在深蹲区(恰好是需求 14 N·m 的最差姿态)。

**判定方法**:
1. **已知力矩自洽验证**:单关节吊空挂已知重物(质量 m、力臂 L),下发
   τ = m·g·L 应恰好悬停——一次性验证 Kt、减速比、单位、方向整条换算链;
2. `robot_web.py` 方向验证功能逐关节确认发力方向(标定后必做);
3. 核对 `robot_web.py` 起身插值的中间姿态:用 `dynamics.py` 算插值路径上
   逐点的力矩需求,确认轨迹是否长时间停留在 >12 N·m 的深蹲区。

**判据**:已知重物悬停实验偏差 >20% → 换算链有错,逐级查;方向验证有反向 →
改 `joint_map` / `stand_config`。

### H3 仿真-真实失配(仿真侧"作弊"了)

仿真里 12 N·m 够用,可能是仿真低估了需求:

- **质量失配**:USD 标称 11.70 kg 是 CAD 值,真机含线缆/电池/外壳可能更重;
- **质心偏移**:电池位置等导致前后腿负载不均,最重的那条腿先到上限;
- 仿真摩擦/接触模型偏乐观;闭链连杆传动效率(齿轮啮合 + 四连杆)未计入
  (仿真直接驱动等效关节,真机经齿轮副有效率损耗,η≈0.9 则 12→10.8);
- 仿真没建 `armature`(转子惯量×N²≈40×),动态需求被低估(对静态站立影响小,
  对"起身加速段"有影响)。

**判定方法**:
1. **称重**:整机实际重量 vs 11.70 kg,差多少需求按比例放大;
2. `standup_torque.py` 跑同款起身轨迹,导出逐关节力矩 CSV/曲线,与真机
   电机 FB 力矩(或电流×Kt×N 推算)**同姿态对比**——差距大的关节就是失配点;
3. 四腿分别测静立时的支撑力(秤垫脚下),看质心是否偏。

**判据**:实重超标称 >10% → 改 USD 质量重训/降低站姿需求;同姿态仿真 vs
真机力矩差 >30% → 摩擦/效率失配,在仿真侧补效率系数或 DR 覆盖。

### H4 电机本体到头(最后查,已被选型分析基本排除)

峰值 23.7 N·m、余量 1.78×,理论上够。剩余风险只有:温升降额(连续堵转在
深蹲姿态,电机过热限流)、个别电机性能不良。

**判定方法**:单电机堵转测力矩(测力计×力臂),抽查 2-3 个;红外测温枪看
起身失败后电机温度。

## 4. 推荐排查顺序(决策树)

```text
单电机吊空,前馈力矩逐档加到额定
 ├─ 达不到额定 ──→ H1 驱动器参数 / H4 电机本体(读寄存器→堵转测试)
 └─ 能达到
     ├─ 已知重物悬停实验偏差大 ──→ H2 换算链(Kt/减速比/单位/方向)
     └─ 换算链 OK
         ├─ 整机发力时母线掉压 ──→ H1 供电(电池/分电板/线径)
         └─ 电压稳
             └─ H3 失配:称重 + standup_torque.py 同姿态力矩对比
                 └─ 顺带必做:优化起身轨迹避开深蹲区 + 站立加重力前馈
                    (无论根因是哪个,这两条都直接降低需求侧压力)
```

## 5. 需求侧优化(与排查并行,降低对力矩的要求)

1. **起身轨迹避开深蹲区**:深蹲(h≈0.256)需求 14 N·m 是站定(2.2)的 6 倍,
   改用「先收腿到力臂小的姿态再垂直撑起」的轨迹,峰值需求可大幅下降
   (用 `dynamics.py` 对候选轨迹逐点算力矩,选峰值最小的);
2. **站立重力前馈**:纯 PD 扛重力浪费力矩裕度,加 τ_ff = 重力补偿项;
3. **抬高站姿目标**:站定高度 0.32 比蹲姿 0.256 的保持力矩小得多,
   RL 侧同样要修(`base_height` 权重,见 todo §评估发现)。
