# 四足机器人「站不起来」原因分析与排查

> 适用：`motor_control/scripts/robot_web.py` 操控网页 + `motor_ctrl`（GO-M8010-6 电机，减速比 6.33）。
> 配套工具：网页里的**扭矩监控图**（实时力矩+峰值+上限线）、**角度跟踪图**（当前角−站姿目标）、**输出日志**。

---

## 0. 先按现象分类

| 现象 | 最可能的方向 | 跳到 |
|------|-------------|------|
| 软绵绵、一掰就动、撑不住自重 | K_P 太低 | [§1](#1-kp-刚度不够最常见) |
| 站到一半突然掉下去 | 峰值力矩不足 / 掉电 / 保护跳闸 | [§2](#2-扭矩不足过载) |
| 好几个关节完全不动 | 分段机制 / K_P 不足 / 某总线没驱动 | [§5](#5-分段抬升机制看起来没动) [§7](#7-servo-进程--总线问题) |
| 点站立直接被拒绝、根本不动 | 安全校验未过 / 缺配置 / 无响应 | [§6](#6-安全校验拒绝) |
| 角度差了约 56.9° 的整数倍 | 单圈编码器圈数掉电清零 | [§8](#8-单圈绝对编码器的圈数问题) |
| 抖动、过冲、力矩乱跳 | K_P 太高 / K_W 太小 / 双重驱动 | [§1](#1-kp-刚度不够最常见) [§7](#7-servo-进程--总线问题) |

## 1. K_P 刚度不够（最常见）

关节保持/抬升的力矩近似：

```
τ = τ_ff(前馈) + K_P·(目标位置 − 当前位置) + K_W·(0 − 当前角速度)
```

**抬升与保持的力气几乎全来自 `K_P × 位置误差`**。K_P 太低 → 出力太小 → 撑不住自重、抬不起来、一掰就动。

- 手册 `Kpos` 范围：**0 ≤ K_P ≤ 25.599**（界面超出会自动夹到上限并红框提示）。
- 经验值：
  - `K_P = 2`：悬空验证用的"温柔"值，**正式站立太软**（会一掰就动）。
  - `K_P = 8~15`：正式站立常用区间。
  - `K_P = 25`（当前默认）：接近上限、很硬，有劲但**容易抖动/过冲/力矩冲高**。
- `K_W`（阻尼）：默认 0.1。抖动时适当加大（0.5~2）压震。

**判断**：站到位后看**角度跟踪图**——若某关节柱子明显偏离 0、或一掰它柱子变大且不回弹，就是刚度不足，调大 K_P。

> 注意：界面 K_P 框可填超 25.599，但实际被夹到 25.599（框会变橙提示真实值）。别误以为填 80 就有 80。

## 2. 扭矩不足 / 过载

从趴到站，**中段腿最伸展、力臂最大、需要的力矩峰值最高**，所以"站到一半掉下去"很典型。

- 反馈力矩 `τ`（FB 的 `data.T`）是**转子侧 N·m**；关节输出端 = 转子侧 × 6.33。
- 手册只给了协议**编码上限 ±127.99 N·m**（非物理额定）；物理额定需查电机规格/堵转实测。扭矩监控图默认上限 23.7 仅为常见参考，**请按实际改**。

**判断（扭矩监控图）**：
- 某关节峰值柱**顶到/超过上限红虚线**、同时**跟踪误差仍大** → 真·扭矩不足（出满力也顶不住）。
- thigh / shank 承重最大，最先到上限；行程最大的关节（如本机 `fr_thigh` Δ≈4.34rad）也最吃力。

**对策**：
1. 调慢：`速度`调小、`最小时长`加大 → 降低动态峰值力矩。
2. 加**前馈力矩**补偿自重（见 §3）。
3. 用**分段抬升**（见 §5）降低同时抬升的峰值电流。
4. 若确实超物理额定 → 减负载 / 换更大力矩关节。

**其他"突然掉"的诱因**：
- **电源/电池欠压（brownout）**：中段多电机同时大电流把母线拉垮 → 电机复位释放 → 整体软掉。量起身瞬间母线电压，用够大电源/满电。
- **过温/过流保护**：FB 的 `temp` 近 90℃ 触发过热保护，`merr`(MERROR)=1 过热 / 2 过流；保护后需重新上电。

## 3. 前馈力矩 τ_ff

纯 PD 靠"位置误差"才出力，误差小时出力不足 → 下沉。前馈力矩**不靠误差、直接给恒定力矩抵消自重**，能在误差很小时也顶住。

- 界面两处：「整机站立参数」的**前馈力矩**（给所有关节的统一基准，±8 N·m 转子侧），与每关节表里的 `t_ff`（单独微调），二者**叠加**。
- 站立时按抬起进度 **0→满** 施加，保持阶段维持。
- **符号必须与"顶住自重"方向一致**，反了会帮倒忙（加速下沉甚至蹬腿）。
- 从小往大调（±0.2→0.5→1.0…），边看跟踪误差边加。
- ⚠️ 单位是**转子侧** N·m，8 转子 ≈ 50 关节侧，很大；先用小值。

## 4. 速度 / 最小时长（动态力矩）

实际运动时长 = `max(最小时长, 最大位移 ÷ 速度上限)`。

- `速度`太大 / `最小时长`太小 → 中段加速度大 → 瞬时力矩过冲、易触发保护或失步。
- 起不来 / 掉下去时，**先调慢**（速度 0.05、最小时长 3~5s）降低峰值力矩再试。

---

## 5. 分段抬升机制（"看起来没动"）

当前 `staged=True`。站立分 **3 段**：`STAGE_ORDER = 小腿 shank → 大腿 thigh → hip`。

- 每段只抬升 **4 个关节（每条腿一个）**，**其余 8 个原地保持不动**，等自己那一段。
- 所以**任意时刻只有 4 个关节在动**，另外 8 个"没动"是**设计如此**，稍后会动。
- 日志会有三行 `[分段] 抬升 shank/thigh/hip（…），其余关节保持。`
- 想一次全动：去掉「分段抬升」勾选（整机同时）。
- 目的：降低同时抬升的峰值力矩/电流，缓解 §2 的掉电/过载。

**判断"是否真没动"**：等三段全跑完再看；若某关节始终没到位，再按 §1/§7 区分是"使劲顶不动(扭矩不足)"还是"压根没驱动(总线问题)"。

## 6. 安全校验拒绝

点站立会先读当前角，与配置**趴姿**比对：`|当前 − 趴姿|`（已按整圈归一，见 §8）超过 `校验阈值`（默认 0.5rad）就**拒绝执行**。

- 现象：点了没反应 + 日志/弹窗提示"安全校验未通过 / 缺配置 / 无响应"。
- 原因：**没在趴姿上电/开始站立**、或某关节离趴姿太远、或电机不在电机模式。
- 对策：让机器人自然趴好再点站立；确认电机已进电机模式、接线正常。
- `hard_bad`（配置缺失 / 无响应）**不能**被"关闭安全校验"绕过，必须先解决。

---

## 7. servo 进程 / 总线问题

- **双重驱动**：电机还锁位保持时再点站立，旧 servo 进程若没停掉，会和新进程**同时驱同一批电机** → 反馈乱跳、力矩异常。
  （已加保护：新动作开始前自动停掉仍在运行的旧 servo；保险起见先点「⏹ 急停/松开」再重新站。）
- **管道断开 / 进程崩溃**：日志出现 `[警告] servo /dev/ttyUSBx 管道已断开` → 那一路总线的电机失令松开。
- **某条腿整条不动**：一条腿 = 一路总线（ttyUSB0~3，各 3 个电机）。该总线 servo 没起来 / 接线松 → 这 3 个关节都不动。

**判断（扭矩监控图）**：没动的关节若**力矩≈0** = 没被驱动（查总线/接线/日志）；**力矩很高** = 在使劲但扭矩不足（§1/§2）。

## 8. 单圈绝对编码器的圈数问题

编码器是**转子侧单圈绝对**：

- **单圈内绝对位置**：掉电保留（不动就不变）。
- **圈数（多圈累加）**：掉电**清零**，每次上电从第 0 圈算。

后果：同一物理位置，上电后读数可能比上次差**整数个转子圈**：

```
1 转子圈 = 360° ÷ 6.33 ≈ 56.9°（关节）
```

界面里所有「(°)」都是关节角（已 ÷6.33），所以这种偏移以 **~56.9° 的整数倍**出现（常被看成"差了 50°"）。

- **站立本身不受影响**：目标 = `当前读数 + Δ`（Δ=站−趴，同一次标定测得的相对量），相对量与圈数无关。
- **影响的是安全校验**：偏差差一整圈会超阈值被拒。已修复——校验偏差按整圈归一到 ±π 再比阈值（日志会提示"差 ±N 个转子整圈"）。
- **前提**：必须**在趴姿附近上电**（误差 < 半圈 ≈28°），归一才能就近对正确的圈。
- 趴姿零点别压在单圈边界（0/2π）上，否则上电读数会在 0/2π 间抖。

## 9. 用界面三件套快速定位

| 没动/掉下去的关节表现 | 角度跟踪图 | 扭矩监控图 | 结论 → 对策 |
|----|----|----|----|
| 在使劲顶不动 | 差值柱大、不归 0 | 力矩高 / 顶到上限 | **扭矩不足** → 调大 K_P、调慢、加前馈、分段、减负载 |
| 没被驱动 | 差值柱大 | 力矩 ≈ 0 | **总线/进程问题** → 查日志`管道已断开/无响应`、查该腿接线 |
| 软、晃 | 差值柱中等、随手晃 | 力矩小 | **K_P 太低** → 调大 K_P |
| 抖、过冲 | 柱在 0 附近抖 | 力矩大幅摆动 | **K_P 太高/K_W 太小** → 降 K_P、加 K_W |
| 其实到位了 | 差值柱 ≈ 0 | — | 正常 |

**日志重点看**：`[轨迹]…`（参数/时长）、`[分段] 抬升 …`（分段顺序）、`[拒绝]…`（安全校验）、`[警告] … 管道已断开`、`差 ±N 个转子整圈`。

## 10. 当前配置与推荐起点

当前 `config/stand_config.json` 的 `execute`：

```json
{ "verify_threshold_rad": 0.5, "max_joint_vel_rad_s": 0.05,
  "min_duration_s": 2.0, "rate_hz": 100,
  "k_p": 25, "k_w": 0.1, "staged": true }
```

**推荐排查/调参顺序（机器人架空进行）**：

1. **K_P**：先 10 试稳；太软加大（到 15~25），抖就回调到 8~12。`K_W` 抖了加到 0.5~2。
2. **速度/时长**：起不来或掉下去 → 速度 0.05、最小时长 3~5s（降动态力矩）。
3. **前馈**：仍下沉 → 给承重关节（thigh/shank）小前馈（±0.3~1，符号对准撑起方向）。
4. **分段**：开着能降峰值电流；想一次全动可关。
5. **上电姿势**：务必趴姿上电再站立（避开单圈边界）。
6. **电源**：确认母线电压在大电流时不塌、电池满电。
7. 全程盯**扭矩图**（别超物理额定/过温）和**角度跟踪图**（是否到位）。

> 永久改参数：编辑 `stand_config.json` 的 `execute` 段后点网页「📂 重新加载配置」；
> 临时本次生效：直接在「整机站立参数」框里填（留空=用配置默认）。
