# 站立功能 —— 真机调试方案

> 目标：在真机上安全地把「站立」从底层伺服到整机起身逐级验证。**原则：自底向上、每级一个
> 通过门槛(gate)，不过不进下一级；任何异常立刻急停。** 最危险的是新写的 `servo` 流式位置
> 伺服，所以先在「单电机手持/空载」上验证它，再碰整机。

---

## 0. 准备与安全约定

**硬件 / 场地**
- [ ] 准备**吊装点或支架**：Stage 4 前能把机器人整机吊起、足端离地。
- [ ] 一人**随时手按急停**（网页 `⏹ 急停/松开`）+ 物理断电开关在手边。
- [ ] 场地无人无障碍，线缆不会被关节绞到。

**软件 / 连通**
- [ ] `motor_ctrl` 已编译：`ls motor_control/Linux/build/motor_ctrl`
- [ ] 4 路串口在位：`ls /dev/ttyUSB*` 应有 `ttyUSB0..3`
- [ ] `sudo` 免密策略 OK（脚本里 `SUDO_PASSWORD="1"`，与系统一致）

**调参约定（先慢先软）**
- 调试期把 `config/stand_config.json` 的 `execute` 改成**更保守**值，过门槛后再逐步恢复：
  - `k_p: 2.0`（先很软，能用手掰动）、`k_w: 0.5`
  - `max_joint_vel_rad_s: 0.05`（更慢）、`min_duration_s: 2.0`
  - `verify_threshold_rad: 0.20`（先松一点，确认能放行后再收紧）

**通用中止**：任何阶段电机异响/抖动/朝错方向/温度报警 → 立刻急停 + 断电，记录现象再排查。

> 在本会话执行并把输出带回，或把输出贴给我，我帮你判读。

## Stage 0 — 连通 / 映射 / 电机模式（不动电机）

**做什么**
1. 先用电机调试页确认每路总线上的 ID 和我们的映射一致：
   `python3 scripts/motor_web.py` → 「扫描全部 ttyUSB*」。
   期望：usb0=1,2,3 / usb1=4,5,6 / usb2=7,8,9 / usb3=10,11,12。
2. 确认所有电机在**电机模式**（不是工厂模式，绿灯不是每秒快闪3次）；若有工厂模式的，点
   「切回全部 ttyUSB*」。
3. 启动操控页 `python3 scripts/robot_web.py` → 「📐 读取当前 12 关节角」。

**Gate 0（必须全过）**
- [ ] 扫描出的 ID/总线**完全匹配** `joint_map.default.json`（不匹配就改配置或改接线）。
- [ ] 「读取当前 12 关节角」**12 个全部 ok**（表里 current 都有值、无 ⚠）。
- [ ] 连读 2~3 次，同一关节读数**基本稳定**（漂移 < 0.01 rad），说明通信稳。

> 不过的话先解决：少电机=接线/ID/供电；某路全无=该 USB 没枚举或线序反。

## Stage 1 — `servo` 流式伺服：单电机手持/空载（验证新代码）★最关键

**目的**：在不连累整机的前提下，验证 `motor_ctrl servo`：能收 stdin 目标、位置环跟随、
**方向正确**、**很慢**、退出补停止脉冲；并验证 `sudo -S` 喂密后 stdin 仍能持续流式写入。

**做法**（拿一个**空载或可手持**的电机，例如先单独接 usb0 的 id=1）
1. 读当前转子角：`sudo motor_control/Linux/build/motor_ctrl /dev/ttyUSB0 1 read`
   记下 `rotor=R`。
2. 先「原地保持」测试（应纹丝不动、能感到弱保持力，kp 很低）：
   ```bash
   ( printf '1 %s\n' "$R"; sleep 3; echo stop ) | \
     sudo motor_control/Linux/build/motor_ctrl /dev/ttyUSB0 servo 2.0 0.5
   ```
3. 「缓慢小幅移动」测试：把目标从 R 线性升到 R+0.10（约 1s，低增益），观察方向与速度：
   ```bash
   ( R=<填上一步的R>
     for p in $(seq 0 20); do
       printf '1 %s\n' "$(python3 -c "print($R + $p*0.005)")"; sleep 0.05
     done
     sleep 2; echo stop
   ) | sudo motor_control/Linux/build/motor_ctrl /dev/ttyUSB0 servo 2.0 0.5
   ```
   再把 `+ $p*0.005` 改成 `- $p*0.005` 验证反方向。

**Gate 1（必须全过）**
- [ ] 保持测试：电机**不乱转**，stdout 周期打印 `servo id=1 tgt=.. Pos=.. ok=1`。
- [ ] 小幅移动：电机朝**目标增大方向**缓慢转约 0.10 rad（Pos 跟到 R+0.1 附近），**不超调猛冲**。
- [ ] 反向同样正确。
- [ ] 进程结束（echo stop / Ctrl-C）后能看到 `servo stop pulse` 与 `servo exit`，电机**松开**。
- [ ] 全程通过 `sudo` 仍能持续把多行喂进 stdin（验证管道没被 sudo 截断）。

> 这一关过了，说明新固件链路 + 流式协议没问题，后面整机才敢动。

## Stage 2 — 标定流程 + 安全校验逻辑（机器吊起/侧卧，腿可自由动）

**目的**：验证网页标定三步、配置文件内容、以及第4步安全校验（不放行/放行）逻辑，**此时不
真正起身**或仅极小幅度。

**做法**
1. 机器人**吊起或侧卧**，让腿能自由活动、不会真支撑体重。
2. 摆成「趴」姿 → 点 ① 记录趴姿；手扶摆成「站」姿 → 点 ② 记录站姿；点 ③ 保存。
3. 打开 `config/stand_config.json`，逐关节核对：
   - `delta_rotor = stand_rotor - prone_rotor`，符号(方向)与你预期的"趴→站每个关节怎么转"一致；
   - 量级合理（关节角 = Δ/6.33，换算成度看是否符合直觉）。
4. **校验「不放行」**：把某条腿明显掰离趴姿，点「站立」→ 应被**拒绝**并红色标出该关节、
   日志写明偏差>阈值。
5. **校验「放行」**：摆回趴姿，再点站立 → 应**通过校验**进入执行（此时已吊起，看方向）。

**Gate 2**
- [ ] 三步标定 12 关节齐全，`stand_config.json` 生成、`delta` 符号与量级合理。
- [ ] 故意偏离 → 站立被拒，且指出正确的关节。
- [ ] 摆回趴姿 → 校验通过、能进入执行阶段。

## Stage 3 — 吊起状态下的站立轨迹（看方向、看协调、看限速）

**目的**：整机吊起、足端离地，跑完整站立插值，确认**每个关节朝正确方向、同步到位、足够慢**，
但不承重、摔了也没事。

**做法**（保守参数：kp=2, vmax=0.05, min_dur=2）
1. 吊起，摆趴姿，点「站立」，全程手放急停。
2. 观察：12 关节是否都朝"趴→站"方向动；是否**同时到位**（限速使最大位移关节恰好到 vmax）；
   末态是否**保持**住（servo 持续重发目标）；点「急停/松开」后是否松开。
3. 用日志里的 `[轨迹] ... 时长 Xs` 核对：`时长 ≈ 最大关节位移 / vmax`。

**Gate 3**
- [ ] 所有关节方向正确、无某关节反向。
- [ ] 运动缓慢平滑、无猛冲/抖动；到位后稳定保持。
- [ ] 急停能即时停下并松开。
- [ ] 若某关节反向 → 多半是该电机装配/极性导致 delta 方向相反：复查标定或在配置里修正
  该关节（重标定优先，不要手改数值除非确知）。

## Stage 4 — 落地 / 真站立（有人保护 → 独立）

**目的**：逐步加载体重完成真实站立。

**做法（逐步加 kp / 减时长，一步一停）**
1. **半承重**：吊带略放松，让腿部分受力，跑站立；若软塌则**小步**调高 `k_p`（如 2→4→6→8）。
2. **有人扶站**：吊带基本松开，一人扶住机身防倾倒，跑站立，观察能否撑住体重并保持。
3. **独立站立**：撤手（人仍在旁边、手在急停上），完整站立并保持。
4. 站稳后逐步把 `max_joint_vel_rad_s` 调回目标 0.1、`min_duration_s` 调回 1.0，确认仍稳。

**Gate 4**
- [ ] 能撑起体重站立并稳定保持，无明显塌陷/抖振/过热(看日志 Temp、MError)。
- [ ] 急停可靠。
- [ ] 把最终 `k_p/k_w/vmax/min_dur/threshold` 写回 `config/stand_config.json` 默认值并记录。

## 调参速查

| 现象 | 处理 |
|------|------|
| 站起来软塌、撑不住 | 调高 `k_p`（小步：2→4→6→8…），必要时加 `k_w` |
| 抖动/啸叫 | `k_p` 偏高或 `k_w` 偏低 → 降 `k_p` 或加 `k_w` |
| 起身太快不安全 | 降 `max_joint_vel_rad_s`（0.1→0.05），或加 `min_duration_s` |
| 某关节反方向 | 重做标定；确认该电机 prone/stand 读数顺序没反；接线/极性 |
| 站立总被拒 | 没摆回趴姿（当前角离配置趴姿太远）；或机械改动过未重标；必要时临时放宽 `verify_threshold_rad` 排查 |
| 读不到角度 | 工厂模式(swmotor 切回)/接线/供电/ID 冲突 |
| 站完不松手保持中想释放 | 点「急停/松开」 |

## 常见排查命令

```bash
# 单电机读角
sudo motor_control/Linux/build/motor_ctrl /dev/ttyUSB0 1 read
# 兜底杀死所有 servo/驱动子进程
sudo pkill -f motor_ctrl
# 看某路总线有哪些 ID（用电机调试页扫描，或）
sudo motor_control/Linux/build/motor_ctrl /dev/ttyUSB0 all read
```
