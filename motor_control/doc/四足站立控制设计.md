# 四足机器人操控网页 —— 站立功能实现文档（设计稿，待确认）

> 目标：做一个类似手柄的网页操控面板，先只实现「站立」一个动作。前进/后退/坐下等
> 后续动作沿用同一框架扩展。本文档先梳理思路，确认后再写代码。

## 1. 范围与目标

- **本期只做**：站立（趴 → 站）一个动作的完整闭环：标定 → 保存配置 → 安全校验 → 执行。
- **界面形态**：仿手柄的操控面板，但本期按钮只放「站立」（其余前进/后退/站立切换等做成
  占位按钮或后续接入）。
- **新建独立网页** `motor_control/scripts/robot_web.py`，与现有电机调试页
  `motor_web.py` 分开，但复用它已经验证过的串口扫描 / 子进程 / sudo / 轮询日志框架。

非目标（本期不做）：行走步态、IMU/姿态反馈闭环、力控柔顺、自动判断"扶住到位"。

## 2. 硬件与系统现状（已确认）

- 机器人 **12 自由度**：4 条腿 `FL / FR / RL / RR`，每腿 3 关节 `hip / thigh / shank`。
- 电机：宇树 **GO-M8010-6**，减速比 `GEAR_RATIO = 6.33`，`关节角 = 转子角 / 6.33`。
- **总线拓扑**：每条腿一路 USB-RS485 总线，共 **4 路** `/dev/ttyUSB0..3`，每路挂 3 个电机，
  腿内 ID 约定为 `0=hip, 1=thigh, 2=shank`（实际 ID 在配置里可改）。
- 现有 `motor_ctrl`（C++）支持三种动作：
  - `drive`（速度环）、`stop`（mode=0 刹车）、`read`（mode=1 + 全 0 增益，只读不驱动）。
  - **不支持位置控制** —— 站立需要按位置把关节移动到目标角，所以要给 `motor_ctrl` **新增
    一个位置控制子命令**（详见 §6）。
- 电机命令字段（来自 `unitreeMotor.h`）：`mode`(0 刹车/1 FOC闭环)、`T` 力矩、`W` 速度、
  `Pos` 目标转子位置(rad)、`K_P` 刚度(0~25.599)、`K_W` 阻尼(0~25.599)。位置控制即
  `mode=1, K_P>0, K_W>0, Pos=目标转子角, W=0, T=0`。

> ℹ️ 电机为**单圈绝对值编码器**：`Pos` 的绝对位置**断电后不变**，因此标定出的
> `prone_rotor / stand_rotor / delta_rotor` **跨上电周期持续有效**，无需每次上电重标；
> 仅在机械改动 / 重新装配 / 换电机 / 改 ID 后才需重标。
> （边界：单圈绝对 + 减速比 6.33，若某关节 趴→站 的转子行程超过一整圈 ≈ 6.28 rad
> ≈ 0.99 rad 关节角，存在圈数歧义；此时第4步安全校验会因当前趴姿读数对不上而拒绝并提示，
> 不会误动。）

## 3. 总体流程（对应你给的 4 步）

```
┌── 标定阶段（一次性；单圈绝对编码器，标定后跨上电有效）──────────────┐
│ 1. 趴姿标定：机器人自然趴下，读取全部 12 个关节当前角 → prone_pose      │
│ 2. 站姿标定：人手扶着把机器人撑成站立姿态，读取全部 12 个关节角 → stand_pose │
│ 3. 计算 趴→站 每个关节的转动量 delta = stand - prone（带符号=方向），      │
│    连同 prone_pose 一起存成配置文件 stand_config.json                  │
└──────────────────────────────────────────────────────────┘
┌── 执行阶段（运行时，可多次）──────────────────────────────────┐
│ 4. 安全校验：重新读当前 12 关节角，逐个与配置里的 prone_pose 比较，         │
│    全部 |current - prone| < 阈值 才放行；否则报警拒绝执行。              │
│    放行后：目标 = current + delta，沿插值轨迹平滑发位置指令完成站立。      │
└──────────────────────────────────────────────────────────┘
```

要点：
- 执行时目标基于**当前实测 prone 角**再加 delta（`current + delta`），而不是直接用标定时
  的 `stand_pose` 绝对值——把动作绑定到「本次校验通过的当前趴姿」上更安全（也正好用上你
  要求的"角度差 + 方向"）；绝对编码器下二者数值上接近，用 `current + delta` 更稳健。
- delta 的**符号**就是方向；执行时按符号把关节朝该方向转 `|delta|`。

## 4. 配置文件格式

路径：`motor_control/config/stand_config.json`（新建 `config/` 目录）。

```jsonc
{
  "robot": "mos2026_2",
  "gear_ratio": 6.33,
  "calibrated_at": "2026-05-29T15:00:00",   // 标定时间戳（仅记录，不强制重标）
  "encoder_note": "单圈绝对值编码器，绝对位置断电不变，配置跨上电有效",
  "joints": [
    // 每个关节一条；unit 统一用「转子角 rad」（与 motor_ctrl 的 Pos 一致），
    // 展示时再 /6.33 转关节角、*180/pi 转角度。
    {
      "name": "fl_hip",  "port": "/dev/ttyUSB0", "id": 0,
      "prone_rotor": 1.234,   // 第1步读到的趴姿转子角
      "stand_rotor": 3.456,   // 第2步读到的站姿转子角
      "delta_rotor": 2.222    // 第3步 = stand - prone（带符号）
    },
    { "name": "fl_thigh", "port": "/dev/ttyUSB0", "id": 1, "...": "..." },
    { "name": "fl_shank", "port": "/dev/ttyUSB0", "id": 2, "...": "..." }
    // ... FR(ttyUSB1) / RL(ttyUSB2) / RR(ttyUSB3) 各 3 个，共 12 条
  ],
  "execute": {
    "verify_threshold_rad": 0.20,   // 第4步安全校验阈值（转子角 rad，约 1.8° 关节角）
    "max_joint_vel_rad_s": 0.1,     // ★关节角速度上限：缓慢安全起身（governing 约束）
    "min_duration_s": 1.0,          // 时长下限（即使位移很小也不至于太快）
    "rate_hz": 100,                 // 流式发指令频率
    "k_p": 8.0,                     // 位置环刚度（保守起步，需现场调）
    "k_w": 0.8                      // 位置环阻尼
  }
}
```

**关节↔(port,id) 映射表**单独放一个默认模板 `config/joint_map.default.json`，
界面首次标定时按它生成 `joints` 列表（用户也可在页面上调整 ID）。

## 6. `motor_ctrl` 新增位置控制子命令（C++）

- **读取角度**：复用现有 `motor_ctrl <port> <id> read` —— 它已经输出转子角
  `ANGLE id=.. rotor=.. joint=.. deg=..`（`rotor=data.Pos`）。标定 / 安全校验直接用它，
  **无需改动**。

- **流式位置伺服（新增）**：100Hz × 12 电机若每 tick 都 spawn 一个子进程不现实，因此给
  `motor_ctrl` 新增一个**常驻流式模式**，每路总线开一个进程，从 **stdin** 接收目标位置：

  ```
  motor_ctrl <port> servo <kp> <kw>
    # 循环以固定频率对本总线电机发 mode=1,K_P=kp,K_W=kw,Pos=<最近目标> 的位置指令；
    # 后台线程非阻塞读 stdin，每行: "<id> <pos_rotor> [<id> <pos_rotor> ...]" 更新目标；
    # 无新行时持续重发上一次目标 => 到位后保持（holding）；
    # 收到一行 "stop" 或 SIGTERM/EOF => 发 mode=0 停止脉冲后退出。
  ```

  - **插值仍在 Python**（满足你"差值先 python"的要求）：后端按 `rate_hz` 计算每个关节的
    目标位置，把同一总线 3 个关节拼成一行写进对应 servo 进程的 stdin。
  - 这样 C++ 只管底层 FOC 收发，Python 管轨迹；4 路总线各一进程、可并行。

改完需在 SDK 目录重新 `cd build && cmake .. && make motor_ctrl`。

## 7. 网页后端设计（`robot_web.py`）

复用 `motor_web.py` 的：`Controller` 子进程/日志框架、`sudo -S` 喂密、`/api/state`
轮询、串口列举。**新增**机器人级状态机与接口。

### 状态机

```
IDLE → (标定趴姿) → PRONE_DONE → (标定站姿) → STAND_DONE → (保存) → CONFIGURED
CONFIGURED → (站立: 先校验) → VERIFYING → (通过) EXECUTING → STANDING
                                        → (不通过) → 报警, 回 CONFIGURED
```

### HTTP 接口

| 方法 | 路径 | 作用 |
|------|------|------|
| GET  | `/`               | 操控面板页面 |
| GET  | `/api/state?since=`| 轮询：状态机、日志、最近读到的 12 关节角、配置是否就绪 |
| GET  | `/api/config`     | 返回当前 `stand_config.json` |
| POST | `/api/calib/prone`| 第1步：读全部关节角，存入内存 `prone_pose` |
| POST | `/api/calib/stand`| 第2步：读全部关节角，存入内存 `stand_pose` |
| POST | `/api/calib/save` | 第3步：算 delta，写 `stand_config.json` |
| POST | `/api/stand`      | 第4步：安全校验 → 通过则执行插值站立 |
| POST | `/api/sit` (占位) | 反向回趴（后续） |
| POST | `/api/estop`      | 急停：杀所有子进程 + 对全部关节发 stop 脉冲 |

### 读全部 12 关节角

按 `joints` 列表对每个 `(port, id)` 跑 `motor_ctrl <port> <id> read`，用现有
`parse_angle_line` 解析出 `rotor`，汇总成 12 条记录。按总线分 4 个线程并行（腿内 3 个
串行），一次性读取，用于标定与安全校验。

### 站立执行（Python 速度限幅插值 + C++ 流式伺服）

```python
# 伪代码
cfg = load_config()
cur = read_all_rotor()                          # 第4步先读当前转子角
for j in cfg.joints:                            # 安全校验：必须贴近配置趴姿
    if abs(cur[j] - j.prone_rotor) > cfg.verify_threshold_rad:
        abort("关节 %s 偏离趴姿 %.3f > 阈值，拒绝站立")
targets = {j: cur[j] + j.delta_rotor for j in cfg.joints}

# ★关节角速度限幅：算出"最慢"总时长，使位移最大的关节恰好 = max_joint_vel
vmax_rotor = cfg.max_joint_vel_rad_s * GEAR_RATIO          # 关节限速换算到转子
max_move   = max(abs(targets[j] - cur[j]) for j in cfg.joints)
duration   = max(cfg.min_duration_s, max_move / vmax_rotor)
# => 每个关节速度 = |delta_j|/duration ≤ max_joint_vel，所有关节同时到位

# 4 路总线各开一个 servo 进程
servo = {port: spawn("motor_ctrl", port, "servo", cfg.k_p, cfg.k_w) for port in ports}
N = int(duration * cfg.rate_hz)
for k in range(1, N+1):
    a = k / N                                   # 0→1 线性（可换 ease-in-out 更柔）
    for port in ports:                          # 每总线拼一行: "id pos id pos id pos"
        line = " ".join(f"{j.id} {cur[j] + a*(targets[j]-cur[j]):.5f}"
                        for j in joints_on(port))
        servo[port].stdin.write(line + "\n")
    sleep(1/cfg.rate_hz)
# 到位后不退出：servo 进程持续重发末态目标 => 保持站立。急停/坐下时再 stop。
```

> 说明：限速由 `max_joint_vel_rad_s=0.1` 主导，`min_duration_s` 仅作下限。例如位移最大的
> 关节需转 0.5 rad（关节角），则 `duration = 0.5 / 0.1 = 5s`，缓慢起身。

## 8. 网页前端设计（手柄风格）

- 顶部：串口状态、配置状态徽章（未标定 / 已标定 / 站立中）、急停大红按钮（常驻）。
- **标定区**（折叠/向导式，3 步）：
  1. `① 记录趴姿`（读角，表格显示 12 关节趴姿角）
  2. `② 记录站姿`（手扶撑起后点，显示 12 关节站姿角）
  3. `③ 计算并保存配置`（显示 delta 表 + 方向箭头，写 json）
- **操控区**（手柄样式）：中间方向键（前/后/左/右，本期置灰占位）+ 中央
  `🧍 站立` 主按钮（仅在 `CONFIGURED` 且安全校验思路下可用），`🪑 坐下` 占位。
- **关节表**：实时显示 12 关节 当前角 / 趴姿 / 站姿 / delta / 校验偏差，超阈值标红。
- **日志窗**：复用 `motor_web.py` 的黑底滚动日志。

交互安全：点「站立」先弹确认框（提示"确保机器人趴好、周围无人/无障碍"）；执行中除急停外
其余按钮禁用；校验不通过用红色横幅说明哪个关节超差。

## 9. 安全设计

1. **执行前安全校验**（你的第4步）：当前角必须全部贴近配置趴姿，阈值可配，默认 0.20 rad
   转子角（≈1.8° 关节角）。任一超差即拒绝并指明关节。
2. **关节限速 + 平滑插值**：关节角速度上限 `0.1 rad/s`（缓慢起身，主导约束），线性插值
   （可改 ease-in-out），`K_P` 保守起步（8.0），现场可调。
3. **急停**：任何时刻可杀全部子进程并对 12 关节发 `stop` 脉冲。
4. **绝对编码器**：单圈绝对值编码器，绝对位置断电不变，标定配置跨上电有效，无需每次上电
   重标；仅机械改动 / 重装 / 换电机 / 改 ID 后重标。站立前的安全校验仍是独立的安全门槛。
5. **默认只监听 127.0.0.1**，与 `motor_web.py` 一致，避免暴露到局域网误触。
6. **工厂模式检查**：若电机处于工厂(boot)模式读不到角，提示先用 `motor_web.py` 的
   `swmotor` 切回电机模式。
